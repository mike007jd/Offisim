# Phase 2.4 — Production Hardening Design

> **Goal:** Eliminate Anthropic hard-binding, enable OpenAI-compatible providers, validate with real LLM calls.

## 1. Problem Statement

The core runtime has 103 tests but has never been validated against a real LLM. The user has keys for three providers — none of which the current code can use:

| Provider | Key | Current Support |
|----------|-----|----------------|
| Gemini | `GEMINI_API_KEY` | ❌ No adapter |
| Kimi | `KIMI_API_KEY` at `https://api.kimi.com/coding/v1` | ❌ OpenAiAdapter lacks `baseURL` |
| OpenRouter | `OPENROUTER_API_KEY` at `https://openrouter.ai/api/v1` | ❌ Same |

Additional issues:
- `LlmProvider` type only allows `'anthropic' | 'openai'`
- `ModelResolver` hardcodes `anthropic` + `claude-sonnet-4-20250514` as fallback
- `graph.smoke.ts` only works with Anthropic key
- No `.env.example` documentation

## 2. Industry Research

Studied [Vercel AI SDK](https://ai-sdk.dev/providers/openai-compatible-providers) (70k+ ⭐) and [LiteLLM](https://github.com/BerriAI/litellm) (38.2k ⭐):

**Key patterns adopted:**
- Factory function pattern (`createGateway(config)`) — from Vercel's `createOpenAICompatible`
- `baseURL` + `apiKey` as minimum config for any OpenAI-compatible endpoint
- Single `'openai-compat'` provider type for all compatible endpoints (not per-provider enums)

**Patterns NOT adopted:**
- `provider/model` string convention (LiteLLM) — would require refactoring ModelProfile, ResolvedModel, event payloads
- `transformRequestBody` hook — none of our 3 providers need it
- Native Gemini SDK — Gemini has an OpenAI-compatible endpoint, reuse OpenAiAdapter

## 3. Architecture Changes

### 3.1 OpenAiAdapter Constructor

```typescript
// BEFORE:
constructor(apiKey: string, retryConfig?: RetryConfig)

// AFTER:
export interface OpenAiAdapterOptions {
  baseURL?: string;
  defaultHeaders?: Record<string, string>;
  retryConfig?: RetryConfig;
}

constructor(apiKey: string, options?: OpenAiAdapterOptions)
```

Changes:
- `new OpenAI({ apiKey, baseURL, defaultHeaders })` — OpenAI SDK natively supports these
- `providerLabel` stored as `options?.baseURL ? 'openai-compat' : 'openai'` for error mapping
- `stream_options: { include_usage: true }` conditionally omitted when `baseURL` is set (not all compat endpoints support it)
- When usage is unavailable, token count records as 0. Phase 3 UI will show a notice.

### 3.2 LlmProvider Type Extension

```typescript
// packages/shared-types/src/models.ts
// BEFORE:
type LlmProvider = 'anthropic' | 'openai';

// AFTER:
type LlmProvider = 'anthropic' | 'openai' | 'openai-compat';
```

`'openai-compat'` covers Gemini (via compat endpoint), Kimi, OpenRouter, Ollama, LM Studio, and any future OpenAI-compatible provider.

### 3.3 createGateway Factory

New file: `packages/core/src/llm/gateway-factory.ts`

```typescript
export interface GatewayConfig {
  provider: LlmProvider;
  apiKey: string;
  baseURL?: string;
  defaultHeaders?: Record<string, string>;
}

export function createGateway(config: GatewayConfig): LlmGateway {
  switch (config.provider) {
    case 'anthropic':
      return new AnthropicAdapter(config.apiKey);
    case 'openai':
      return new OpenAiAdapter(config.apiKey);
    case 'openai-compat':
      if (!config.baseURL) {
        throw new Error('openai-compat provider requires baseURL');
      }
      return new OpenAiAdapter(config.apiKey, {
        baseURL: config.baseURL,
        defaultHeaders: config.defaultHeaders,
      });
  }
}
```

### 3.4 ModelResolver De-hardcoding

```typescript
// BEFORE:
const HARDCODED_DEFAULT: ResolvedModel = {
  provider: 'anthropic',
  model: 'claude-sonnet-4-20250514',
  temperature: 0.7,
  maxTokens: 4096,
};

constructor(policy: ModelPolicyConfig | null | undefined)

// AFTER:
const SYSTEM_FALLBACK: ResolvedModel = {
  provider: 'openai-compat',
  model: 'default',
  temperature: 0.7,
  maxTokens: 4096,
};

constructor(
  policy: ModelPolicyConfig | null | undefined,
  fallback?: ResolvedModel,
)
```

- `fallback` overrides `SYSTEM_FALLBACK` when provided
- `SYSTEM_FALLBACK` is intentionally generic — forces callers to provide a policy or explicit fallback
- Existing callers that pass a valid policy are unaffected
- Test fixtures provide explicit model policy, so tests don't hit fallback

## 7. Decisions Log

| Decision | Rationale |
|----------|-----------|
| No native GeminiAdapter | Gemini has OpenAI-compat endpoint; avoids new `@google/generative-ai` dep |
| `'openai-compat'` single enum value | Avoids enum explosion; baseURL differentiates endpoints |
| `stream_options` conditionally omitted for compat | Not all compat endpoints support it; token=0 acceptable with Phase 3 UI notice |
| No `transformRequestBody` hook | None of 3 current providers need it; add in Phase 3 if needed |
| No `provider/model` string convention | Would require refactoring ModelProfile, ResolvedModel, events; too large for Phase 2.4 |
| ModelResolver fallback is generic `openai-compat/default` | Forces explicit policy config; Anthropic no longer privileged |
