## Positioning (locked with user)

This is the frame every action below is measured against:

- **Core employee runtime** (maintained, finite, architecturally load-bearing): `packages/core/src/llm/anthropic-adapter.ts`, `openai-adapter.ts`, `subscription-adapter.ts` (ACP via `claude` subprocess). These drive LangGraph employee nodes. **Not touched by this audit.**
- **External-agent interop extension points** (sleeping, open-ended, architecturally not load-bearing): `packages/core/src/a2a/` + `packages/core/src/gateway/openclaw-client.ts`. These exist as code-only hedges for future agent runtimes. **Not wired to any UI, not instantiated anywhere, not promoted in any doc.** If 2.0 decides to enable external agent interop, the preferred path is A2A (see Phase 6 roadmap). Until then, both directories are dormant.
- **What counts as "mockup"**: only user-facing UI that pretends to provide a working feature while the backing logic is fake. Silent backend code that's never reached by users is NOT mockup — it's just unwired code.
