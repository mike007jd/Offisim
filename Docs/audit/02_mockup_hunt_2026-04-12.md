## CRITICAL — production-path mockup, must fix before ship

### C1. Settings → Gateway tab (OpenClaw) is 100% mock, but real client already exists
**Severity: CRITICAL SHIP BLOCKER**

- UI entry: `packages/ui-office/src/components/settings/SettingsTabNav.tsx:9` registers a "Gateway" tab with key `'openclaw'`.
- Tab renders `OpenClawSettings` component at `packages/ui-office/src/components/openclaw/OpenClawSettings.tsx:10,44`, which calls `useOpenClaw()` from `packages/ui-office/src/hooks/useOpenClaw.ts`.
- **The hook is entirely mock**:
  - `useOpenClaw.ts:119-123` — `async function mockConnect(url, token)` sleeps 800ms and always resolves successfully. `url`/`token` unused (prefixed `_`).
  - `useOpenClaw.ts:86-111` — hard-coded `MOCK_AGENTS: OpenClawAgent[]` with 3 fake agents:
    - `{ id: 'oc-1', name: 'CodeReviewer', model: 'claude-3.5-sonnet', skills: ['code-review', 'refactoring'] }`
    - `{ id: 'oc-2', name: 'Translator', model: 'gpt-4o' }`
    - `{ id: 'oc-3', name: 'DataAnalyst', model: 'claude-3-haiku', status: 'offline' }`
  - `useOpenClaw.ts:113-117` — `MOCK_GATEWAY_INFO = { version: '0.9.0-mock', uptime: '2h 14m', agentCount: 3 }`.
  - `useOpenClaw.ts:142-158` — `connect()` calls `mockConnect`, then `setAgents(MOCK_AGENTS)` + `setGatewayInfo(MOCK_GATEWAY_INFO)` + `setConnectionState('connected')`.
  - `useOpenClaw.ts:169-194` — `inviteAgent()` is **not mock**: it emits a real `employeeInstalled(activeCompanyId, agentId, ...)` event on the runtime event bus. This puts a ghost lobster employee into the live scene backed by no real gateway session.
- **Real client already exists**: `packages/core/src/gateway/openclaw-client.ts` defines `OpenClawClient extends TypedEmitter` — a full WebSocket client for OpenClaw Gateway Protocol v3 with `connect()`, `listAgents()`, `sendChat()`. It's re-exported in `packages/core/src/browser.ts:286-290`. The hook comment at line 83 says "Mock client — replace with real OpenClawClient import when available" but **real client is already available**.
- **User-visible failure mode**: User fills in the Gateway URL + token fields in Settings, clicks Connect → gets a fake "Connected" state + 3 imaginary agents. Clicks "Invite CodeReviewer" → a lobster puppet appears in the office scene, clicking it or chatting with it produces no real agent response because there's no session.
- **Not mentioned in CLAUDE.md or MEMORY.md** — this is an unflagged regression in the audit surface.

**Decision the user owns**: (a) wire the hook to real `OpenClawClient` (Phase 3-sized work), (b) hide the Gateway tab entirely for 1.0, (c) ship as-is with "Preview" label. Phase 5 will not pick for you.

### C2. `useInstallFlow.ts` has 3 MOCK_INSTALL_PLAN fallback branches in production path
**Severity: HIGH (borderline CRITICAL pending path analysis)**

- File: `packages/ui-office/src/hooks/useInstallFlow.ts`
- Imports: `import { MOCK_INSTALL_PLAN } from '../lib/install-mock.js'` (line 21). This is **not a test file** — it's in `src/lib/`.
- 3 fallback sites, all structured as `if (!installService) { setTimeout 500ms → setPlan(MOCK_INSTALL_PLAN); setStep('review') }`:
  - `:129-138` — `beginPackageImport` (the core flow used by `startFileImport` for `.offisimpkg`/`.zip` drops)
  - `:185-194` — SKILL.md import path
  - `:275-285` — Upgrade flow
- **Supporting mock data**: `packages/ui-office/src/lib/install-mock.ts` is a **src-tree file, not a test fixture**. It exports `MOCK_MANIFEST` + `MOCK_BINDING_REQUIREMENTS` + `MOCK_INSTALL_PLAN` hard-coded as:
  - package: `offisim.employee.writer_pro` v1.2.0 "Writer Pro" by `alice` / "Alice"
  - `source_url: 'https://market.example/packages/writer-pro-1.2.0.offisimpkg'`
  - `integrity.package_sha256: '9f2c9f2c...'` repeating pattern
  - Declared as `Track D will replace mock usage with real InstallService calls` (file header line 4) — Track D is never mentioned in CLAUDE.md or MEMORY.md.
- **Conditions for triggering**: `installService` is null in these code paths:
  - `apps/web/src/runtime/BootstrapProvider.tsx:79` — `installService: null` during initial bootstrap before runtime is ready
  - `apps/web/src/lib/tauri-runtime-lite.ts:43` — `installService: null` in lite runtime mode
  - `packages/ui-office/src/runtime/offisim-runtime-context.tsx:42` — comment says `null in Tauri mode or when runtime is not yet ready` — **note: the "Tauri mode" part of this comment is STALE DRIFT**, see D2 below.
- **Real install service construction** (confirmed not null in normal runtimes):
  - `apps/web/src/lib/tauri-runtime.ts:282` — `const installService = new InstallService({...})` ✓
  - `apps/web/src/lib/browser-runtime.ts:277` — `const installService = new InstallService({...})` ✓
- **User-visible failure mode**: In the window between first boot and runtime-ready, if the user drags a `.offisimpkg` file onto the install dialog, they see the Writer Pro fake plan (alice, MIT, 128304 bytes) and can click through the review step — at which point the install button either errors (if install service is still null at confirm time) or materializes nothing.
- **Why it exists**: historical dev-aid fallback before install service was wired. Now purely vestigial.

**Recommendation** (Phase 5 action): Replace all 3 fallbacks with `setStep('error'); setError('Install service not ready — retry once runtime is loaded')`. Delete `install-mock.ts` and remove the import. Delete the `Track D` legacy comments.

### H2. `a2a-server.ts` `tasks/get` not implemented — A2A async mode broken
**Severity: HIGH (depends on product A2A positioning)**

- `packages/core/src/a2a/a2a-server.ts:210-218`:
  ```ts

    // Current implementation is synchronous (blocking mode only).
    // tasks/get for async task tracking will be implemented when we add
    // non-blocking server-side execution with persistent task storage.
    const taskId = (rpc.params as Record<string, unknown>).id as string;

  ```

- `a2a-server.ts:4` header: `Lets external agents (e.g. OpenClaw lobsters) send tasks TO Offisim employees`. So A2A is designed as a *server*-side integration point.
- **Product question** the user owns: Is A2A async mode in the 1.0 promise? If yes, CRITICAL. If A2A is blocking-only by design or pre-1.0, this is a SUSPECT-allowed gap.
