State & conditional rules
  • Right region content: selectedEmployee == null → ONE shared .empty filling the whole 1fr region (never a blank reserved column); else Details header + 6-tab inspector.
  • Roster: employeesLoading → skeleton rows; employeesError → page error + Retry (refreshEmployees()); employees.length===0 → Hire + Browse; filtered-only empty → Reset (title/CTA switch on employees.length===0 vs filter mismatch).
  • Roster collapse: tier≠narrow && useSidebarCollapse('personnel') collapsed && rows>0 → 56px avatar-only, name sr-only + title={row.name}; search/filter hidden while collapsed.
  • Save bar visibility = isDirty only; Save enabled when isDirty && name.trim() && !isSaving; while saving → "Saving…"; failure → inline red + Retry. Delete is an inline confirm inside the bar; deleteError surfaces inline.
  • Profile model-mode toggle: Inherit vs Custom — Custom reveals model family + override id + provider-scoped suggestion chips + max-tokens (thinking-model ≥1024 warning).
  • Runtime: engine cards disabled when adapter unavailable or profile blocked; resolved line shows the effective route; external → single Lock line, no rebind.
  • Appearance: external (isExternal) → no customizer, fixed brand previews; no-selection inside a tab → shared empty (handled at region level here).
  • Skills: company skill shadowed by an employee skill of the same slug → "overridden by your own" tag; empty → quiet dashed/empty.
  • History: rollback 2-step ("Rollback to vN?" → Confirm/Cancel, "Rolling back…"); diff add=green / remove=red / change=amber; no-history / single-version / no-selection quiet states.
  • Tablet: roster narrows, inspector still in the single right region (no third column ever). Narrow: list when no selection, detail+tabs when selected, header Back clears selection.
Reverse-risk register
  • Old IA defect — a near-empty 1fr center spacer sat between the roster and a separate 420px inspector, and that inspector column rendered blank on no-selection. This rewrite deletes the center column entirely: roster | Details+Inspector (1fr), one shared empty state. Do not reintroduce a 3-column / reserved-empty layout.
  • Old IA defect — Skills appeared twice (Profile▸Config mini-list AND the Skills tab) and there were N independent per-tab save bars. This rewrite enforces ONE Skills locus (Skills tab) and ONE detail-level save bar. Do not re-embed Skills in Profile or add per-tab save bars.
  • Live ProfileTab.tsx ships the model-mode toggle labels in Chinese (跟随统一设置 / 自定义模型) — violates the ui-office "UI all-English" rule. This spec renders the corrected English; fix at source, do not localize chrome ad hoc.
  • Live VersionHistoryTab / VersionDiffTable still use legacy ocean palette classes (text-shell/50 · bg-sea-blue · text-kelp-green · text-lobster-red · text-coral-orange · border-ocean-light) — off the token system. This spec maps them to neutral/info/ok/danger/warn; migrate to design tokens at source.
  • Engine boundary (CLAUDE.md 2026-05-09): file/shell/memory/skill/MCP/builtin tools must NOT be exposed via an unverified model transport. Per-employee "Model binding" text engines are model-only; gateway-tools & SDK-native full-agent profiles unlock only after deterministic + release-app + denied-path + cancellation + rollback + sandbox + credential-boundary evidence passes.
  • "Runtime" name collision (§3): the Personnel per-employee control is "Model binding"; the company-scoped one is Settings ▸ Runtime "Default employee model". Never put three "runtime"s on one screen.
  • External A2A peers are brand-avatar outsourced employees, not an "external department" (that historical abstraction was reverted). They take a seat/zone/ceremony like internal employees; only 2D/3D rendering branches by brandKey, with no DiceBear/block-person reuse.
