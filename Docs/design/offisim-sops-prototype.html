State & conditional rules
  • Single step-edit entry. The ONE primary path is node double-click → editor / add-step popover (submit reads "Save"). The right-click context menu's "Edit" and the on-demand panel's "Edit step" button mirror the same flow — they are not parallel primary affordances. double-click blank canvas opens the same popover in Add mode at the click point. Documented here per IA §5; only the one primary entry is shown as primary in chrome.
  • Node detail occupancy. Panel is mounted only when selectedStepId != null; it overlays the canvas right edge and the canvas keeps full width underneath. Closing (×, Esc, or deselect) removes it and reclaims the space. It is never a reserved/blank column.
  • Ports. Render every frame. Outside Edit mode: opacity ~0.38 + pointer-events:none. In Edit mode: input port = accent, output port = warning; valid hover target → success accept ring; cycle/self-loop drop → danger reject ring + silent abort. ⎋ cancels an in-progress connection. Edge mid-point × hit area ≈ 14px (specimen 1).
  • Edge stroke by upstream status. pending = line-strong 2px · active = accent 2.5px + traveling dot · completed = ok 2px · failed = danger 2px (short-circuits downstream). (specimen 5)
  • Run progress strip. Mounts when the live plan matches this SOP and a run is active, and persists ~3s after finish. Stop is the same single abort surface as the status-bar Stop (one logical control, surfaced where the run is visible — not a third control).
  • Save status. Mutations float a save pill at the canvas top-right: "Saving…" / "All changes saved" / "Save failed" (specimen 5).
  • Role-missing run guard. A step whose role has no enabled employee shows the ⚠ missing-role chip + a warning in the node detail. Run is NOT blocked — the dispatcher falls back; the guard is informational, matching product behavior.
  • Auto-layout. Topo-sort batch layout assigns columns by dependency depth; port positions recompute. "Fit" is the single fit-to-view affordance (was a separate floating control in the old surface).
  • Responsive. narrow tier: sidebar becomes a drawer, node detail becomes a bottom sheet, Edit mode is disabled (touch). tablet: sidebar collapses to the first-letter rail (shown in the resting window's collapsed structure). Per IA, narrow verification target is the web SPA in browser, not the desktop shell.
Reverse-risk register
  • The live SopInspectorPanel is read-only and previously earned a permanent third column even when nothing was selected. This rewrite makes it on-demand overlay only — implementer must NOT restore a persistent inspector column; the canvas owns that width.
  • The old surface exposed step editing through 4 parallel affordances shown as primary (toolbar Add Step, dblclick blank, dblclick node, right-click menu). Keep all paths in code but present only the node double-click as the primary; the rest mirror it. Do not re-promote them to competing primaries.
  • Old file had 3 Create-SOP entries (sidebar +, empty hero, none-selected hero) and 2 Import entries (sidebar download, empty hero). Consolidate creation/import to the ONE sidebar-header locus; the empty-state hero buttons reuse that same flow, they are not independent loci.
  • Old "SopRunProgressStrip" + status-bar Stop could read as two abort controls. They are one logical action; do not add a third Stop anywhere.
  • Old appendix shipped a dark "STATE SPECIMENS 1:1 with SopViewSurface" block with code-name captions exposed to users. Specimens here are clean designed visuals with plain captions; all code refs live in this drawer only.
  • SOP "synced" indicator reflects registry/marketplace sync state, not local save. Do not conflate it with the save pill.
