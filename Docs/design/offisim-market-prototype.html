Component map (visible block → real component)
  • Resting grid card → MarketListingCard (MarketCardGrid.tsx); ONE style, drawn once in grid + once in split detail context. The old 20× "rarity matrix" legend is deleted (its full mapping is in "Kind→tone" below).
  • Topbar Explore/Manage segment + chip rows → MarketFilterBar.tsx (replaces the 3 sibling dropdowns Mode/View/Kind + 2 oversized <select> + Publish button).
  • Split right pane → MarketDetailView.tsx rendered inside WorkspaceRouter (non-Office, side panel null, no right rail). It is the ONLY detail surface.

  • Install sequence dialogs → InstallDialog = ManifestReview → BindingForm → UpgradePreview (upgrade path) → InstallProgress → DoneContent / ErrorContent, all in DialogShell.
  • Publish dialog → PublishDialog (DialogShell xl).
  • Deep-link overlay → MarketplaceDetailOverlay — legacy, kept ONLY for offisim://install?listing_id=X.
Kind → tone mapping (real product tokens, no "lie" comment)
  • Source: market-rarity.ts getRarityColor(kind). The collapsed shared palette maps each app rarity to a real status token:
  • employee · bundle → --accent (info identity, shared blue).
  • skill · company_template → --violet (the distinct "accent" rarity survives the collapsed palette as violet).
  • sop · prefab → --warn.
  • office_layout → --danger (its "error" rarity tone).
  • unknown / DEFAULT_RARITY → --ink-3 neutral.
  • Tone applies to kind chip text, cover-icon tint, and detail kind pill — never as a literal hex; all via the .k-* token classes. No CSS comment claims a "lie".
State & conditional rules
  • INSTALLABLE_KINDS = {employee, skill} — only these render the Install CTA. All other kinds render "Install not supported for {kind}." (specimen 2).
  • Installed = per-company installedListingIds ∪ installedPackageKeys; card Installed badge shows only when installed && INSTALLABLE_KINDS.has(kind); detail CTA becomes disabled "Installed".
  • Detail open: showDetail; desktop/tablet = workspace split (grid stays left, MarketDetailView right). Narrow (web SPA ≤768) = full-page drill-in, filter bar hidden, Back returns to grid (specimen 11).
  • Loading detail = back btn pinned + skeleton (3fr/2fr). Unavailable = "Listing unavailable" + Back + a "Listing unavailable" toast (specimen 3).
  • Explore loading = 8 skeleton cards (SKELETON_CARD_KEYS). Error (401/503/transport) = MarketErrorState with technical detail + Retry + optional cached-count hint. No results = MarketEmptyState + Reset filters (specimen 9).
  • Manage → Installed filtered to install_state==='installed'; per-item Check update disabled when no origin_listing_id; Update btn when origin_listing_id && hasUpdate; Updates tab adds the uppercase sub-header (specimens 4–5).
  • Published state machine unauth | loading | ready | error (auth from RegistryClient.hasAuthToken); statuses Draft / Validated / Submitted / Approved / Rejected (specimen 6).
  • Install entry points (detail CTA / deep link / file import) all funnel into the ONE InstallDialog sequence. BindingForm renders only model_profile bindings (mcp_endpoint / secret_slot future); required+empty → inline "Required"; optional → Skip/Configure.
  • Upgrade path swaps ManifestReview for UpgradePreview (version from→to, change counts, migration notice, breaking alert, category-grouped diff) then InstallProgress (specimen 8).
  • PublishDialog Kind select shown only when active company has both employees AND skills; Cancel-while-dirty → shared discard-confirm toast; "Submit draft" sends platform-verified artifact bytes (specimen 10).
  • Installed-state updates are install-event-driven: an install/upgrade success event re-derives the company installed set; grid badges + detail CTA recompute without manual refetch.
Reverse-risk register
  • Prior pass shipped a CSS comment calling the rarity palette a "lie" and rendered the card ~20× as a rarity-matrix legend + a "1:1 with MarketPage" dark appendix. Removed: the mapping now lives here only; the card appears once in grid + once in split detail.
  • Detail was rebuilt 3× (split / panel / narrow page) with 4 install entry points. Consolidated to ONE MarketDetailView (workspace split; narrow is the same view at layout="narrow") and ONE InstallDialog funnel. Deep-link overlay is legacy-only.
  • Mode/View/Kind were 3 sibling dropdowns + 2 oversized 1/5-width <select>. Now: Explore/Manage = one topbar segment; Kind+Sort = compact chip rows; Manage sub-views = one tab row. No full-width selects.
  • "Listing unavailable" surfaces both an in-pane state AND a toast — keep both wired (do not silently drop the toast on the redesign).
  • Non-Office route correctly has NO right rail; do not re-introduce a collab rail on Market just because the shell defines one.
