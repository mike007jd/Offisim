State & conditional rules
  • API-key warning + Settings CTA: rendered ONLY when no provider is configured. Resting product hides it.
  • Direct-chat header (back-to-Team arrow + name): rendered ONLY when a direct employee is selected; team chat has no header — thread switching lives in Inspect ▸ Threads, not a second list.
  • ErrorBanner: Retry / Swap Person (dropdown) / Swap Model / Details (errorCode · node · employee · taskRun · recoverable · timestamp + history last 5) / dismiss.
  • Deliverables render by timestamp inside the assistant message that produced them; new deliverable highlights ~3s + scrolls into view; Save-as-SOP shows "Saved!" 2s; promoted only when ≥2 contributors.
  • Streaming bubble: reasoning collapsible; chunk grows inside the bubble (caret = active stream).
  • InteractionPrompt: severity 'high' → DialogShell modal (no backdrop / Escape close); else inline in the stream; agent_question disables non-cancel options until note is non-empty; skill_install_confirm → SkillInstallConfirmBubble (fork / edit / create variants; wide-scope tool → "Review permissions" badge; frontmatter error → reason + Retry/Cancel).
  • PipelineProgress: 5 nodes Boss→Manager→PM→Employee→Summary; states completed / active / pending / error; connector colors; Stop only while running; secondary label = ceremony || route; persists ~3s after run. Status-bar Stop and PipelineProgress Stop are the SAME single abort action.
  • MeetingPanel renders null when idle (participants / type badge / duration / Live|Ended); MeetingActionItems post-meeting with priority badges + per-item Delegate.
  • Onboarding starter chips: empty-state variant of the message list (no thread activity yet) — hidden in the resting running state.
  • Attachment staging: 4 entry paths (paperclip / paste / drag-drop / context); overlay copy; parsing→parsed chip; 6 verbatim error states; rollback on partial persist failure. ChatContextStrip = project + staged-attachment count above ChatInput.
  • SessionModeChip is the ONLY mode entry: SOP / Human-in-loop / Direct / YOLO; YOLO recolors Send to danger.
  • Inspect file tree (ProjectWorkspaceFiles): desktop sandbox list / bounded preview (64 KB) / refresh / parent-up only; filter is client-side; no create / rename / delete; no per-row git mark in the real tree. Web shows a desktop-only state.
  • Inspect Threads: empty = "No threads yet."; active row = border-focus + accent-muted; dbl-click → inline rename (Enter save / Esc cancel); auto-title rewrites titles via chat_thread.updated (ThreadList / WorkspaceSearch refetch).
  • Inspect Outputs empty = "No Outputs Yet" (shared .empty); no project bound = "Select a project to inspect files, threads, and outputs."
  • Tasks Activity always shown; idle = "Run a task to see live activity here."; same-entry bursts collapse to an ×N badge.
  • Tasks Plan rendered only when planSteps>0 || stage==='planning'; running with no plan → centered stage-label card; no plan + idle → section hidden. Board summary references the one Board chip — no second toggle in Tasks.
  • Kanban: tray Board chip ↔ officeState.kanbanOpen; expanded board = 5 columns (Todo / Doing / Blocked / Review / Done) over the stage; ⎋ / outside-click collapse; invalid transition shows an alert; empty board still renders; card pencil → inline edit; PlanKanbanBoard variant (Requirements + step columns) reuses the same shell.
