State & conditional rules
  • ResumeBar shown only when isOffice && unfinishedThreads.length>0. Hook queries threads.findByCompany(companyId,{status:'running'}) + {status:'blocked'}, enriches with project name. Per-project verb = status==='blocked' ? 'Review' : 'Resume'; onResume(threadId) opens that thread.
  • ResumeBar headline resolution: only blocked → "1 project needs review" / "{n} projects need review"; mixed → "{n} projects need attention"; only running → "1 unfinished project" / "{n} unfinished projects". Chips max-w-[200px] truncate, full name in title. onDismiss clears in-memory detection for the session only — no archive, no persisted "don't show again"; projects.length===0 → return null (no reserved space).
  • ErrorBanner action gates: Retry shown iff onRetry (a failed-run error exists) — reuses the exact same message + config with a new runId; Swap Person iff onSwapPerson && employeeList.length>0 → clearError(), re-adds last user message to the new employee, opens fresh RunScope (new conversationKey + runId); Swap Model iff onSwapModel → onOpenSettings(); Details always (toggles). Options render {agent.name} ({agent.role}).
  • TrackedError = { errorCode, message, recoverable, nodeName, timestamp, employeeId?, taskRunId? }. History capped MAX_ERROR_HISTORY=100; Details shows last 5 reversed only when errorHistory.length>1. Banner never auto-closes — manual Dismiss or a successful new run clears it.
  • PipelineProgress node state per step: idx<active→completed (green ✓), idx===active→active (ring + pulse dot in stage color), idx>active→pending. Stop shown iff isRunning && onAbort → abortExecution(). When isRunning goes false the bar persists ~3 s then setStage(null) — does not snap away. Entirely hidden when compact or no stage. Secondary label = activeCeremonyLabel ?? routeLabel; ROUTE_LABELS: delegate→"Delegating task", meeting→"Starting meeting", direct_reply→"Thinking…", use_sop→"Running SOP", hire_or_assess→"HR assessment".
  • MeetingPanel MeetingState = { meetingId, status, participantIds, startTime, actions[], transcript[] }; subscribes meeting.state.changed + meeting.action.created. status==='idle' → isActive=false → returns null. Badge 'Live' (success) running / 'Paused' (warning) paused / hidden idle; empty (transcript.length===0 && actions.length===0) = "Waiting for discussion..."; duration mono MM:SS; sections render only when non-empty. MeetingActionItems Delegate → onDelegate('@{employeeName} {description}') auto-populates the composer; priority skin high border-error/bg-error-muted, medium warning, low default.
  • Attachment staging error region is aria-live="polite". Exact strings: oversize-per-file "exceeds the 8 MB per-file limit", oversize-total "exceeds the 32 MB total attachment limit", duplicate "Already attached", storage-unavailable "storage unavailable in this browser window", parser-exception "{detail ?? 'parser failed'}", persist-failed "failed to persist attachment". Caps MAX_PER_FILE=8 MB, MAX_TOTAL_BYTES=32 MB; dedupe key name\0size\0sha256 via crypto.subtle.digest. 4 entry points funnel handleStaging(files): paperclip picker, paste (clipboardData.files), HTML drag-drop, Tauri onDragDropEvent (1 s debounce). Overlay copy storageAvailable ? 'Drop to attach' : 'Storage unavailable'; !storageAvailable → paperclip disabled title "Storage unavailable — try a non-private window". Accepted → chat.attachment.staged, rejects → chat.attachment.failed; partial persist failure → rollbackPersistedAttachments. Chip line: "Parsing…" → PDF · {n} pages / {w}×{h} / XLSX · {n} sheets, {n} rows / DOCX · {n} chars / PPTX · {n} slides / Text · {n} chars / Binary file → "Parse error · {reason}"; grid auto-fit minmax(min(100%,11rem),1fr); X → removeStaged(attachmentId).
  • KeyboardShortcutsDialog Cmd/Ctrl + / always toggles (works even with a modal open). The other 5 gated: D/J/1/E require isOffice + empty modal stack; Cmd/Ctrl + E additionally needs officeState.selectedEmployeeId → routeToPersonnel(id,'profile'); Escape closes activeOverlay else workspace-internal back. Slash commands are a separate composer affordance (chat-commands.ts), 15 commands / 5 categories, aliases muted — not keyboard bindings.
Reverse-risk register
  • Drop PipelineProgress → the running pipeline becomes invisible and there is no Stop; the user cannot tell who is running or abort it. Highest-value loss — this is why the resting frame at the top IS the running pipeline, not a buried specimen.
  • Drop ErrorBanner recovery actions → a failed run dead-ends with no Retry / Swap Person / Swap Model / Details path. The banner intentionally never auto-closes; do not "tidy" it away on a timer.
  • Drop ResumeBar → an interrupted/blocked run is unrecoverable after reload (threads stay running/blocked but are otherwise unreachable).
  • Drop the attachment error region → every cap / dedupe / storage / parse / persist rejection becomes a silent no-op; the user sees nothing.
  • Drop MeetingActionItems Delegate → meeting outcomes never convert to delegated follow-up; the meeting becomes a dead-end transcript.
  • error pipeline node state is defined (getNodeState() → bg-error-muted + ✗) but is currently unreached by the live stage mapper. Specimen 03's error pane is intentionally designed so a rebuild wires failure into the bar rather than re-inventing it — do not delete the state because the current mapper never hits it.
  • The 5-stage pipeline (chat bar) and the 8-phase ceremony (3D scene) are independent state machines; the bar's secondary line prefers the ceremony label when both are live. A rebuild must keep them decoupled — collapsing them into one enum loses the scene animation phases.
  • Old states file leaked source-file names as visible captions, repeated the resting window verbatim as a specimen, spent a half-pane on a "renders null" note, and interleaved code/condition prose as primary content. This rewrite relocates all of that here; the rendered page is plain-caption only.
