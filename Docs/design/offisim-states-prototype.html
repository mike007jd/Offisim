Runtime · recovery · global states — state specimens

This is the suite's cross-cutting state layer — the surfaces that wrap every workspace and are the first things dropped when a UI is rebuilt: cross-session resume, chat failure recovery, running-pipeline control, meeting + action items, attachment staging/error, keyboard shortcuts. The window above is the canonical runtime state: Office mid-run, PipelineProgress lives diegetically on the stage content area (V3 §9 — game content area, not chrome) at the executing stage with the inline Stop control. Every specimen below is built 1:1 with the real component and its controlling hook — no invented controls, no dropped states. Each pane is captioned with its source file.

Why this file exists. The other 7 suite files spec each workspace's happy path. The product's hardest-to-rebuild value lives in the not-happy path: how a failed run is recovered, how an interrupted run is resumed across reload, how a running pipeline is read and stopped, how a meeting turns into delegated follow-up, and how attachment failures surface. Those have no single home workspace, so they are specified here as the canonical layer.
