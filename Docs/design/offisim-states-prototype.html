Runtime · recovery · global states — state specimens

This is the suite's cross-cutting state layer — the surfaces that wrap every workspace and are the first things dropped when a UI is rebuilt: cross-session resume, chat failure recovery, running-pipeline control, meeting + action items, attachment staging/error, keyboard shortcuts. The window above is the canonical runtime state: Office mid-run, PipelineProgress pinned above the composer at the executing stage with the live Stop control. Every specimen below is built 1:1 with the real component and its controlling hook — no invented controls, no dropped states. Each pane is captioned with its source file.

Why this file exists. The other 7 suite files spec each workspace's happy path. The product's hardest-to-rebuild value lives in the not-happy path: how a failed run is recovered, how an interrupted run is resumed across reload, how a running pipeline is read and stopped, how a meeting turns into delegated follow-up, and how attachment failures surface. Those have no single home workspace, so they are specified here as the canonical layer.

!Reverse-risk register — what a rebuild must not silently drop
5 rev-risks + 1 note
rev-riskDrop PipelineProgress → running pipeline becomes invisible and there is no Stop; user cannot tell who is running or abort it. Highest-value loss.
rev-riskDrop ErrorBanner recovery actions → a failed run dead-ends; no Retry / Swap person / Swap model / Details path. Banner intentionally never auto-closes.
rev-riskDrop ResumeBar → an interrupted/blocked run is unrecoverable after reload (threads stay running/blocked but are unreachable).
rev-riskDrop the attachment error region → every cap / dedupe / storage / parse rejection becomes a silent no-op; user sees nothing.
rev-riskDrop MeetingActionItems Delegate → meeting outcomes never convert to delegated follow-up; the meeting becomes a dead-end transcript.
noteerror pipeline node state is defined (getNodeState()) but unreached by the live stage mapper — spec keeps it so a rebuild wires failure into the bar rather than re-inventing it.
