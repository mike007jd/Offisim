State & conditional rules
  • getEventLevel (topic only): contains failed/error/rolled_back → Error · blocked/warning/rejected/aborted → Warning · else Info. Info = no border; Warning = border-warning + warn bar; Error = border-error + err bar.
  • collapseConsecutiveReroutes: folds 3+ consecutive task.assignment.rerouted sharing source|reason|taskRunId into one row with ×N; runs <3 stay expanded.
  • domainIcon(type) prefix → icon · colour: default Activity·secondary · hr. UserCheck·error · mcp. Plug·info · knowledge. BookOpen·success · memory. Lightbulb·warning · skill. Puzzle·success · handoff. ArrowRightLeft·accent · meeting./direct.chat. Users·accent.
  • Filters: single-select backed by string arrays — onEventTypesChange(value==='all' ? [] : [value]), same for actors; eventTypes[0] ?? 'all'. Date preset Today / Last 7 days / Last 30 days / All time → today / 7d / 30d / custom (custom = no cutoff). 18 type options = ALL_EVENT_TYPES minus All: Node·Plan·Task·Deliverable·Employee·Install·Skill·LLM·Interaction·Error·MCP·Knowledge·Meeting·HR·Memory·Infrastructure·Git·Attachment. Type label → topic prefixes via TYPE_PREFIX_MAP (Meeting → meeting.+direct.chat.; Infrastructure → rack./slot./binding./cost.). Actors = getAvailableActorFilters(events) (distinct employeeName / name / entityType:entityId, sorted). Search matches type + displayLabel + entityType (lowercased).
  • groupEventsByTime: bucket edges = today 00:00 / yesterday 00:00 / Monday 00:00 / month-1st 00:00. Buckets desc; empty dropped; collapseConsecutiveReroutes applied per group.
  • ActivityEventDetail sections: Event Type (type.replaceAll('.',' / '), mono) · Level badge · Timestamp (formatFullTimestamp, absolute) · Entity (label + optional Type:/ID: mono) · Payload (ActivityPayloadView). Selecting the same row again toggles the panel closed (selectedEventId → null); panel only occupies space while selectedEventId is set — no reserved empty column.
  • Responsive: desktop & tablet identical (default filter bar; split when focused). useLayoutTier()==='narrow' → narrow filter bar (sliders btn + inline search; the 3 controls move into a bottom filter sheet = scrim + bottom panel, focus-trapped, Esc/scrim/X closes via useRegisterModal+useTopmostEscape+useFocusTrap, modal id activity-filter-sheet); on narrow with an event focused the timeline gets hidden so the detail takes the full column. (Per project IA: narrow tier verification target is the web SPA in browser, not the desktop shell.)
  • Hydrating: !bootstrapState && events.length===0 → WorkspacePageSkeleton. no-events: events.length===0 → ActivityEmptyState variant="no-events", Activity icon, primary "Back to Office" (only when onBackToOffice). no-results: filtered to 0 → variant="no-results", Search icon, primary "Reset filters", secondary "Back to Office". Stale selection: selectedEventId set but no matching event (rolled past 200 / filtered out) → info toast "The selected event is no longer available." + reset selectedEventId → null.
Reverse-risk register
  • Reset-filters inconsistency. The no-results "Reset filters" action clears search/types/actors but sets datePreset='30d' — not the default 'today' the workspace boots with. This is a product decision, not a prototype concern: do not invent a new control or change the reset target in the design. Flagged so an implementer either ratifies "reset = 30d" intentionally or aligns it to boot default — owned by product, not this redesign.
  • Single-select masquerading as multi. eventTypes/actorFilters are arrays but the live UI is single-select (['x'] or []). The chip row preserves that semantic (one Type, one Actors value); do not build true multi-select chips without a product decision to widen the model.
  • Bounded store. Only the last 200 events exist client-side; deep history / "Older" bucket is necessarily partial. A selected event can vanish mid-session (stale-selection toast path). Not a bug to "fix" in the prototype.
  • Prior IA defects discarded (do not reintroduce): the timeline redrawn ~6× as a spec appendix; 4 oversized 1/5-width filter selects; raw topic.code.name captions as primary row text; the .rev-flag reset annotation rendered in chrome; a permanently reserved ~420px empty detail column. The redesign humanises labels, uses a compact chip row, and only allocates the detail column on selection.
