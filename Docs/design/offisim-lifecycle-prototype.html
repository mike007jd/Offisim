Global lifecycle

The lifecycle is the cross-cutting layer that wraps every workspace — the surfaces a person passes through from the very first launch to running an everyday office. Each card below is one designed lifecycle state in plain product terms; the calm shell above is the frame these surfaces appear inside.

State & conditional rules
  • First-run gating (App.tsx): welcome shows iff welcome_seen===false && providerConfiguredForWelcome===false && companies.length===0 && tour_dismissed===false. Get started → markWelcomeSeen() only (tour stays armed). Skip and explore / Escape → markWelcomeSeen() + dismissTour(). closeOnBackdrop=false — no backdrop dismiss.
  • Onboarding store localStorage["offisim.onboarding.v2"]: account flags (provider_configured, welcome_seen, tour_completed[], tour_dismissed, first_employee_clicked) + per-company (first_task_sent, first_deliverable_seen). Business flags auto-complete steps via ACCOUNT_FLAG_TO_STEP / COMPANY_FLAG_TO_STEP; v1→v2 migration folds legacy flags once (tour_migrated_v1).
  • Tour step = derived, not stored: activeIndex = first id not in tour_completed[]. Wrong-workspace / no-target → centered card, primary calls onSwitchWorkspace (does not complete step). Last step also dismissTour(). Suppressed when any overlay open or office+directChat.
  • CompanySelectionPage: aside filters status !== 'archived'; selection precedence previewCompanyId ?? activeCompanyId ?? companies[0]. Archive is soft (companies.update(id,{status:'archived'})) — 2-step confirmArchiveCompanyId; if archived was active → onCompanySwitch(null). Empty list / no zones / loading preview states all exist.
  • Company wizard CTA: primaryDisabled = !selectedTemplateId || (!isCreateYourOwn && !runtimeReady) || !companyName.trim(). step = checking | first-run | creating | ready. openingStudio → "Opening Studio…"; isCreateYourOwn → "Open Studio Editor" (→ createCustomCompany() + sessionStorage[PENDING_VIEW_KEY]='studio-edit'); shouldRetryRuntime → "Retry Runtime"; !runtimeReady → "Initializing…"; 'creating' → BuildingAnimation; 'ready' → renders null (unmounts, onComplete(companyId)).
  • ProjectCreateDialog: canSubmit = name.trim && !submitting && (!workspaceRequired || !!workspaceRoot); workspaceRequired = desktopMode && mode==='create'. Web → "Folder binding is desktop-only"; submit failure keeps dialog open; Cancel disabled while submitting.
  • Project status SSOT: planning info · active success · paused warning · completed text-muted · archived text-disabled. ProjectWorkspaceFiles Selection machine: null | loading | ready | error; preview 8 KB request, Rust hardcap 64 KB; no root → "No workspace folder"; web → "Desktop files only".
  • DeliverableCard: body truncate(content,200); format Select only when NOT a file artifact; Save as SOP promoted variant when contributingEmployees.length >= 2; isNew → success glow 3s + auto-scroll; compact card = success-tinted in chat bubble. PitchHall hydrate via listRecentDeliverables({limit:100}) + live deliverable.created, id-first dedupe.
  • EmployeeCreatorOverlay: Back discard-confirm if dirty (name non-empty OR seed manually edited); seed auto-derives from name unless manually edited; deploy disabled until name.trim(); deploy → persona_json.avatarSeed, toast, overlay closes.
  • ExternalEmployeeInstallDialog: Discover enabled when isValidUrl; discoverAgentCard abortable; canConfirm = card & company & repos & eventBus & displayName.trim & roleSlug & !submitting; dirty-close → discard-confirm; Back excluded from dirty.
Reverse-risk register
  • Hard-dark source bug (this prototype's key redesign fix). In source, CompanyCreationWizard and EmployeeCreatorOverlay are intentionally hard-dark — they hardcode text-white / text-slate-* / bg-blue-600 / border-white/[0.06] + a radial dot-grid background, ignoring the light token theme. Cards 3 & 8 are deliberately redrawn in the shared light token system as the corrected target. An implementing agent must migrate these surfaces to tokens deliberately (not keep them off-theme); the divergence is a known decision to resolve, not a faithful spec.
  • InterviewWizard is latent — do not build. An 8-step InterviewWizard (role / name / expertise / style / appearance / instructions / model / preview, hook useInterviewWizard) is exported from @offisim/ui-office but is not mounted anywhere. The live "Add employee" path is the 2-step EmployeeCreatorOverlay (Card 8). Treat InterviewWizard as latent until it is wired; it is intentionally NOT rendered as a specimen here.
  • First-run gating has no user control. Gating is fully derived from onboarding-store flags + company count — there is no "don't show again" checkbox. Do not invent one.
  • Tour steps are derived, not a stored cursor. activeIndex = first incomplete id; business events auto-complete steps independent of the Next button. Do not add a stored "current step" pointer.
