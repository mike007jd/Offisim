State & conditional rules
  • Save bar state machine (the bar label/inline text carries it): no changes → disabled button, "No changes to save"; dirty → "Unsaved changes to <sections>" + active Save; saving → button "Saving…" disabled + inline "applying — runtime will reinitialize"; failed → red inline + button becomes "Retry" + a .save-err banner above content; validation-blocked → amber inline "Resolve N issue(s) before saving" + disabled Save.
  • Save bar is hidden on the External tab (external connections persist per-row, not via the global bar). Topbar "Unsaved" pill mirrors the dirty state and disappears when clean.
  • Credentials field: when access mode ≠ Global API key, the secure-key field is replaced by "Credentials managed by host."
  • Thinking-model warning callout shows only when the selected model variant is a thinking provider.
  • Engine rows: only the Offisim engine is selectable on this host; the 3 transport bindings render disabled ("Unavailable here") — they exist in metadata but cannot run without a verified SDK-native runtime profile.
  • Endpoint override is optional for Global API key, required for access modes that need an explicit endpoint, hidden when the mode resolves it internally.
  • Provider variant row is rendered only when the product exposes >1 variant.
  • Browser vault states: mounted / needs-permission ("Reconnect directory") / error ("Retry mount") / unsupported.
  • MCP add: form error line ("Server name is required"); stdio choice triggers the high-risk confirm card before the server is added; SSE adds without confirm. Empty state: "No MCP servers configured. Add one above to enable tool use."
  • MCP connection states per row: up Connected / down Disconnected (Reconnect) / busy Connecting…
  • External: loading spinner while refreshing; per-row token editor is an inline expand; empty state shown when no agents connected.
  • Catalog refresh error state: "Last refresh failed: network unreachable" replaces the success stats.
Reverse-risk register
  • The source Default employee runtime control was a 2-option <select> carrying an inline apology ("Real control = RuntimeBindingControl, not this select"). Implement the actual RuntimeBindingControl (company-scoped runtime/model profile picker) — the radio-card group here is the design target, not the placeholder.
  • "Lane" was overloaded in the source (provider lane / execution lane / gateway lane). Banned in chrome (§3). Endpoint/transport = "Connection"; driver choices = "Engine". Do not reintroduce "lane" in user-facing text.
  • Source mixed Chinese strings into chrome (模型目录更新 / 上次成功 / 更新模型目录). All chrome must be English; Chinese-string source bug is a localization defect to fix at implementation.
  • Old IA: 4-level nesting (pane→card→section→details→grid) + a 980px cap leaving ~40% dead canvas + 3 competing save buttons. This rewrite enforces max 2 nesting levels, a canvas-filling 210px nav + fluid content with a calm 2-col grid, and exactly ONE save control.
  • Engine boundary (CLAUDE.md 2026-05-09): file/shell/memory/skill/MCP/builtin tools must NOT be exposed via an unverified model transport. The Offisim engine is the only tool-capable path here; SDK transport bindings are model-only unless a verified SDK-native runtime profile + release .app evidence exists.
  • MCP truth source (2026-05-17): client ✓ with stateful tools/resources/prompts/capabilities, but resources/prompts UI surfacing + Streamable HTTP + OAuth are still pending — do not present them as delivered in chrome.
  • Web vs desktop credential storage differs (desktop secure secret file mode 0600, never crosses Rust→JS; web localStorage). Surfaced as a quiet "Secure API key" + destination note in chrome; storage detail stays here, not in chrome.
