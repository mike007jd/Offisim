# V3 Design DNA Brief (internal, do not ship)

Canonical specimen: `offisim-office-layout-v3-prototype.html`. Every other surface
inherits this language. If a rule below conflicts with what an existing prototype
shows, **the prototype is wrong — fix it to match V3**.

## 1 — Philosophy

- **Dense game HUD**, not SaaS marketing.
- Everything on screen is **evidence**: no hero images, no marketing copy, no oversized empty space.
- **Diegetic over chrome**: numbers/state live where they belong inside the surface, not in an OS-style status strip.
- **One axis per region**: never give a region two stacked tab bars.

## 2 — Subtractions (DO NOT reintroduce)

- ❌ Bottom **status bar** with branch / cost / latency / run state.
- ❌ Header **bell icon with count badge** (`.iconbtn .nb`). Notifications are a quiet dot beside diegetic cost readout — not a chrome chip.
- ❌ **Boss Dashboard** entry point. Single-run cost lives in run flow; cross-project ledger deferred.
- ❌ Retired procedure-template workspace/mode/save surfaces. Current execution language is Plan / Human-in-loop / Direct / YOLO.
- ❌ Retired persistent board surfaces. Work state belongs in run status, plan review, activity, and contextual trays, not a separate board system.
- ❌ Right-rail **Chat / Inspect / Tasks tabs**. Right rail is one axis only.
- ❌ Mode chip in **header**. Mode chip lives in the **composer footer**.
- ❌ **`--info` color token as a duplicate of `--accent`**. Always alias `--info → --accent` if both must exist.
- ❌ **Hero image strips** in market / personnel / dashboard cards.

## 4 — Shell (identical across every surface)

Offisim keeps the macOS native titlebar/window controls. The renderer owns the
contained workbench below that native titlebar: a bordered, rounded app surface
on a dark outside frame, with one 54px topbar and no extra renderer titlebar.

```
.app {
  width: calc(100% - 16px);
  height: calc(100vh - 16px);
  margin: 8px auto;
  grid-template-rows: var(--toolbar 54px) 1fr;
  border-radius: var(--r-lg);
  /* native titlebar outside, dark radial outside frame, light bg inside */
}
.topbar    → scope-bar | nav (absolute center) | iconbar
.scope-bar → Company segment + caret > chev > Project segment + caret
.nav       → 4 workspace pills (Office / Workspace / Market / Personnel)
.iconbar   → Activity / Settings only. NO bell. (Office adds a Studio
            iconbtn separated by a 1px divider, after Settings — Studio
            only appears when current workspace is Office.)
```

**Workspace pills** mark the active surface (`active` adds `--accent-surface`
bg + `box-shadow: inset 0 0 0 1px var(--accent-ring)`).

## 5 — Container grammar (the rule)

Every chip / pill / bar that contains buttons follows:

```
height: 30–36px
border: 1px solid var(--line)
border-radius: var(--r-md /* 9px */ or --r-sm /* 7px */ for inner)
background: var(--surface-1) or --surface-2
padding: 3px
inner buttons: 28–30px h, --r-sm radius, transparent bg, hover --surface-sunken
elev-1 default
```

This is the formula behind `.scope-bar`, `.nav`, `.iconbar`, `.ws-tabs`,
`.dock-tools`, segmented controls, filter strips. Reuse it everywhere — never
invent a new tab/segment style.

## 6 — Caps label (the only "section heading" style)

```
font-size: var(--fs-micro /* 10 */)
letter-spacing: 0.14em
font-weight: 680
color: var(--ink-3)
text-transform: uppercase
```

Used for `.sec-head`, `.label`, `.rail-sec-head`, `.gw-label`, `.insp-sec-label`.
**Never** use 14–19px section headers inside body content. Page-level titles
get `--fs-md` or `--fs-lg`.

## 7 — Card grammar (default)

```
border: 1px solid var(--line-soft)
border-radius: var(--r-md /* 9px */)
background: var(--surface-1) or --surface-2
box-shadow: var(--elev-1)
hover: border-color var(--line-strong); box-shadow var(--elev-2); transform: translateY(-1px)
```

For status-tinted cards add a 3px top accent stripe (`border-top: 3px solid var(--accent)` etc.) — see `.kb-col-h`. Do not use a colored left bar except on
DAG nodes (`.dag-node .bar`, 4px wide).

## 8 — Status / pulse vocabulary

```
height: 19–22px; padding: 0 8–11px; border-radius: var(--r-pill)
inner dot: 5–7px circle; .running ::before adds 1.6s pulse animation
font-size: 9.5–11px; weight 700–720; uppercase + 0.05em letter-spacing for status pills
```

Surfaces: `.badge.ok/warn/err/info/violet`, `.run-strip`,
`.dn-status`. Always pair color + surface + animation. Never use the danger
surface for plain hover.

## 9 — Diegetic readouts (replaces status bar)

For surfaces that show a "live scene" (Office), the scene border carries:

```
.scene-cost {
  position: absolute; right: sp-7; bottom: sp-6; z-index: 3;
  height: 26px; backdrop-filter: blur(6px);
  border: 1px solid var(--line); border-radius: var(--r-pill);
  background: rgba(255,255,255,0.82);
}
.scene-cost.live → border-color var(--accent-ring); animated dot
.sc-notif → round 26x26 sibling button; unread shown as dot (no count)
```

Other surfaces (Market/Settings/Personnel/Workspace/Lifecycle) do **not** add this —
they don't have a "live scene" and don't need diegetic numbers.

## 10 — Right rail (Office only)

- Single axis: list ⇄ selected conversation.
- Top: chat-head (48px, back chevron + title + tools).
- Middle: messages flex column gap-6.
- Sediment: `.run-record` collapsible card injected in messages timeline.
- Outputs: `.conv-outputs` block after messages (thread-scoped inline).
- Bottom: composer (input + tool row + mode-chip + send).
- **No tab bar at top of rail.** Ever.

## 11 — Surface-specific guidance

### Office (canonical, locked)
See V3 spec. Do not edit.

### Market
- **Strip SaaS aesthetic.** Cards are **dense inventory chips**, not Steam-store hero cards.
- Card height **≤ 180px**, `--r-md` radius, no image hero by default. Use icon-only cover (44×44 inside a `--rcs` surface, 60px tall band).
- For curated/featured listings only: optional `--r-md` thumb 96×64 (not full-bleed).
- Grid: `repeat(auto-fill, minmax(244px, 1fr))`, gap 14px, padding `sp-7`.
- Filter bar: 48px not 64px; reuse `.nav`-style chip-grammar for kind + sort + mode segmented control. No `select` element with native arrow chrome.
- Detail view: inventory-style route transition. Single-click selects a card; double-click or Enter opens a full-detail view. Do not keep an embedded right-side detail pane beside the grid.
- No `--fs-2xl`. Title `--fs-lg`, summary `--fs-sm`.
- Rarity: alias `--r-employee → accent`, `--r-skill → violet`, `--r-prefab → warn`, `--r-template → violet`, `--r-layout → danger`, `--r-bundle → ink-3`.
- Per-card stats row: stars (warn) · install count · creator handle. Mono font for installs.

### Personnel
- Tri-pane 280|1fr|420 stays.
- Detail head padding `sp-5 sp-7`, not `sp-7 sp-9`.
- Avatar swatch initials kept (matches DiceBear 2D fallback). Keep `.av` radius 26% block-style.
- Inspector tabs use **`.ws-tabs` grammar** (chip-grammar) not loose horizontal buttons.
- Profile field rows: 32–36px input height, 4–6px label gap, no card wrapper around field groups (just caps label + flow).
- Drop the bell.

### Settings
- Left nav 244px keep.
- Right content max-width 720px **inside the pane** (so wide screens don't sprawl). Use a single column of card-grouped sections with caps labels.
- Each section = caps label + `.card-block` (`elev-1` + `--r-md` + `line-soft` border). 16px internal padding.
- Drop the bell.

### Lifecycle
- Wizards (CompanyCreationWizard, EmployeeCreatorOverlay) keep `--wiz-*` dark surface palette **— intentional reverse-risk**, do not relight them.
- All non-wizard dialogs use `.dlg` shell. Dialog padding 16–18px (not 22px).
- Drop the bell.
- Toast / confirm / installable / authorize popovers all use V3 `.icard` skin.

### States
- Composes mini-panes of other surfaces. After other surfaces are refreshed,
  this file just needs its included specimens regenerated.
- Drop the bell.

## 12 — Hard rules (you cannot violate these)

1. **No `--fs-2xl` outside a dialog hero.**
2. **No `--r-xl 18px` on routine cards.** Dialogs only.
3. **No bell + count badge anywhere.**
4. **No tabs on the Office right rail.**
5. **No browser-native dropdown arrow chrome** (use the custom 12×12 chev-down svg
   background like V3 — see `.scope-bar` / `.mkt-sel` for the existing pattern).
6. **No image hero ≥ 80px tall on a card** unless the listing is a curated
   feature card and the height is justified by a content-fitness check.
7. **Caps labels are 9.5–10px, never 12+.** Section headings inside body
   content are uppercase + tracked, not size-only.
8. **Keep every chip/pill/segmented control on the container-grammar formula**
   in §5. Never invent a new visual chrome unit.
