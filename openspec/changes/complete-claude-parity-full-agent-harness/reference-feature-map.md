# Claude Source/Rust Feature Map

## Purpose

This is the source-backed parity map for `complete-claude-parity-full-agent-harness`.
It is the business-logic contract for the features in this change, not a code-copy plan.

Reference roots:

- ClaudeSource: `/Users/haoshengli/Seafile/WebWorkSpace/ClaudeSource/claude-code-haha`
- ClaudeRust: `/Users/haoshengli/Seafile/WebWorkSpace/ClaudeRust/claw-code`

Every feature row must remain mapped to source anchors, Offisim target behavior, and gates.
If a feature has no matching anchor in one reference repo, the row must say so explicitly.
An unanchored feature is not "on the right track"; it is either incomplete or out of scope.

## Use Rules

- Use row IDs in specs, tasks, benchmark scenarios, and implementation PRs.
- Keep rows behavior-level. Do not copy proprietary implementation details, prompts, or UI.
- When ClaudeSource and ClaudeRust differ, the Offisim target column records the product decision.
- A feature cannot be marked complete while its row has a missing gate, unresolved source gap, or stale-truth conflict.
- Stale memory/docs saying a feature is text-only, backend-only, blocked, or future-only are overridden by this map unless the row explicitly marks the route unavailable.

## Coverage Verdict

ClaudeSource is the stronger reference for full coding-agent runtime behavior: main loop,
streaming tool execution, permission prompts, hooks, MCP, subagents, fork/resume, long tasks,
context collapse, and user-visible task output.

ClaudeRust is the stronger reference for local harness discipline: typed runtime surfaces,
file and shell tools, permission enforcement, sandboxing, workspace/session isolation, usage
accounting, and deterministic mock parity harnesses.

Offisim must use both together. ClaudeSource alone under-specs local harness safety. ClaudeRust
alone under-specs full-agent/subagent orchestration.

## Explicit Non-Copy Decisions

- Do not copy Claude UI composition. Offisim should implement the same behavioral obligations in the Offisim product model.
- Do not treat ClaudeRust as complete full-agent parity. It is a local-runtime and mock-harness reference; ClaudeSource is required for full-agent/subagent semantics.
- Do not introduce a generic "SDK lane". SDK-native behavior belongs only to a named full-agent runtime profile or internal transport adapter.
- Do not let browser/dev-server verification satisfy desktop release gates.
- Do not let stale memories or old docs override this map. If old claims contradict a row, update the old claim or mark the row unavailable with evidence.
