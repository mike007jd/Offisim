# Mainstream-Gap Feature Map (G-rows)

## Purpose

Source-backed business-logic contract for `close-harness-mainstream-agent-gaps`. This is the
"borrow the business logic, not the code" map the goal requires. It complements — does not
replace — `complete-claude-parity-full-agent-harness/reference-feature-map.md` (the `F`-rows).
Where a `G`-row refines an already-mapped feature family, it cites the related `F`-row; the
`G`-row carries the concrete, code-level obligation the `F`-row left as a checklist bullet.

## Use Rules

- Use `G`-row IDs in specs, tasks, scenarios, and implementation PRs.
- Rows are behavior-level. Do not copy proprietary prompts/UI/implementation verbatim.
- When the two references differ, the Offisim target column is the product decision.
- A row cannot be marked complete while its gate is missing or its source anchor is unresolved.
- Audit provenance: 2026-05-17 six-domain harness audit. Each row records the Offisim gap anchor.

## Explicit Non-Copy Decisions

- Do not copy ClaudeSource/ClaudeRust source verbatim. Reproduce the behavioral obligation in the Offisim model.
- Do not weaken the existing Rust filesystem/bash sandbox (`apps/desktop/src-tauri/src/builtin_tools.rs`); G02 is an additional front gate, not a replacement.
- Do not delete the org-graph product metaphor for G09; add an isolated sub-run primitive under it and record the divergence.
- Do not let a green deterministic scenario substitute for live evidence where the row still requires live evidence; Anthropic cache-token live smoke is product-descoped for this change and must not be backfilled with fake live evidence.
- Do not enable prompt caching for a provider that does not support it; gate behind a per-provider capability flag (no-op, not error).
